# libl

Runtime companion library for scanners generated by `ft_lex`.

---

## What it provides

| Symbol | Type | Role |
| - | - | - |
| `yywrap()` | function | Called by the scanner at EOF; returning `1` ends scanning |
| `main()` | weak function | Default entry point: opens `argv[1]` or `stdin`, runs the scan loop, calls `yylex_destroy()` |

These are the symbols a generated `lex.yy.c` expects to find at link time but does not define itself.

`input()` and `unput()` are **not** part of `libl` - they live inside the generated `lex.yy.c` itself (in the embedded runtime template), since they need direct access to the scanner's internal buffer state. See "Runtime macros and functions" below.

---

## Usage

```sh
cc lex.yy.c -o scanner -L libl/ -ll
```

The `-ll` flag links against `libl.a`. No `#include` is required in the generated scanner.

---

## Overriding the default `main()`

`main()` is declared with `__attribute__((weak))`. This means the linker will use the `libl` defiunition only if no other `main` symbol is found in the objects being linked. Any `main()` defined in the user code section of the `.l` file (after the second `%%`) or in a separately compiled `.c` file will silently take precedence.

```lex
%%
[a-z]+  { printf("%s\n", yytext); }
%%

int main(void) {
    while (yylex() != 0); /* libl's main() is discarded at link time */
    yylex_destroy();
    return 0;
}
```

## Overriding `yywrap()`

`yywrap()` is a regular (strong) symbol. To override it, define your own `yywrap()` in any translation unit linked before `libl.a`, or simply do not link `libl` at all and provide thge full definition yourself.

A common use case is chainning multiple input files:

```c
int yywrap(void) {
    /* return 0 to signal "more input available", 1 to stop */
    return next_file() ? 0 : 1;
}
```

## Runtime macros and functions (generated into `lex.yy.c`)

These aren't part of `libl.a` itself, but are worth knowing about since they're available in every rule action:

| Name | Role |
| - | - |
| `BEGIN` | Switch start condition. Not function-like (`#define BEGIN yycurrent_state =`) so both `BEGIN(COND);` and the historical bare `BEGIN COND;` compile |
| `REJECT` | Discard the current match and retry the next-highest-priority candidate, even past a `return` on the same line |
| `ECHO` | Write the current token to `yyout` |
| `yymore()` | Append the next match to the current `yytext` instead of replacing it |
| `yyless(n)` | Keep only the first `n` bytes for the current match, pushing the rest back |
| `input()` | Read and consume the next raw byte from the input stream, tracking `yylineno` |
| `unput(c)` | Push a byte into the input stream to the next `input()` (or scan) rereads it, tracking `yylineno` in reverse |

`input()` / `unput()` operate directly on the scanner's internal buffer (growing it as needed), exactly mirroring their POSIX lex semantics.
